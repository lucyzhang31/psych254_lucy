Mechanical Turk

In this class we will be running our replication projects on Amazon's Mechanical Turk (AMT/mTurk). mTurk is a platform on which Workers (or "Turkers") can complete Human Intelligence Tasks (HITs) for monetary compensation. HITs are put up by Requesters. mTurk originally was set up to do large-scale tasks that require human intelligence (e.g. labeling photos, finding telephone numbers, etc), but has recently been used by social scientists to conduct (large-scale) online experiments.

There are many issues to consider when running studies online. Here's just a very short list.

  • You will be reaching a different population (there have been several studies now on Turker demographics) that might be different from "typical" psychological studies (done with WEIRD participants -- white, educated, industrialized, rich and democratic -- see Henrich, Heine, and Norenzayan, 2010).
  • There are constraints on what can or cannot be done -- some psychological manipulations are impossible over the web.
  • Design considerations. Some Turkers multitask, and most of them do HITs for many hours a day, so appropriate designs would include attention checks, manipulation checks, etc.
In this class, all replication projects will be launched using a common class account. This saves you the hassle of applying for a personal Requester account (which lately has become not-as-straightforward...), and also simplifies funding logistics. We will be giving more detailed instructions in class on how to access the class account and launch HITs etc.

Ethics and the Institutional Review Board (IRB)

  • Note that Amazon workerIDs are actually tied to their (public) Amazon account, and thus constitute identifiable information. Thus, you should anonymize data by redacting workerIDs (and other identifiable information).


Best Research Practices

  • Once you collect the data, save the "raw" data somewhere safe, and then commit an anonymized version to version control (Git).
    Do all analyses on the anonymized data. (This is to prevent cases where others are unable to reproduce your analayses because it might rely somehow on identifiable information. If you start with anonymized data, your analyses would never use any of this information.)

    Anonymizing means redacting the workerids (e.g. replacing them with numbers/letters) and other identifiable information (IP address, etc). A quick one liner (in R) is:
    d0$workerid <- match(d0$workerid, unique(sort(d0$workerid)))

  • Document all data transformations using a reproducible script, like an R script. (e.g. converting data from wide form to long form; applying exclusion criteria; etc).
    You should NOT be using Excel to cut and paste to reformat your data. This is NOT reproducible, and is unacceptable for your final project; you should also learn to apply the same mindset to your own research.

Making an external HIT: HTML/Javascript with mTurk

The Computation and Cognition Lab wiki has detailed instructions on how to run external HITs on mTurk. This involves a guide to downloading Amazon's Command Line Tools, writing an experiment, running your experiment, etc. That should be your first stop.

  1. We have a class account for running participants on mTurk.Once you have Amazon's Command Line Tools installed (see Cocolab wiki link above), you will need to modify your mturk.properties file with the access key and password (get this from the TAs).

    This is confidential information which should NOT be put on public version control (i.e. do NOT put your mturk.properties file in a public git repository.) If the credentials gets leaked (e.g. by being on a public repo), then we'll have to re-establish them, making life difficult for many people.

  2. Once you have CLT set up, start making your experiment (see the above wiki for a helpful link, or below for more resources on HTML/Javascript)

  3. Once you have your experiment coded, put it up on the Mechanical Turk Sandbox to test it out. The Sandbox has the same functionality as the "live" mTurk, just that you don't have to pay real money. Because there's no money, there are no "real" participants there, so you needn't worry about people taking your HIT on the Sandbox. The Sandbox is mainly for developers to troubleshoot their HITs.

  4. Get approval from the instructor regarding study design, ethics, power analysis, data analysis plan, etc.
